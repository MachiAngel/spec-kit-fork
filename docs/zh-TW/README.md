## 使用指南

### 新手入門

如果您是第一次接觸 Spec Kit，建議按以下順序閱讀：

1. **第一章** - 了解核心概念和為什麼需要 SDD
2. **第二章** - 安裝和設置 Specify CLI
3. **第七章** - 通過完整範例快速上手
4. **第三章** - 深入了解斜線命令的使用

### 進階開發者

如果您已經熟悉基本概念，想深入了解系統設計：

1. **第四章** - 理解模板系統如何約束 AI 行為
2. **第六章** - 掌握憲章系統的架構原則
3. **第五章** - 了解腳本系統的內部運作

### 團隊導入

如果您計劃在團隊中導入 Spec Kit：

1. **第一章** - 向團隊說明 SDD 的價值
2. **第六章** - 制定團隊的憲章原則
3. **第七章** - 建立團隊的最佳實踐流程

## 文檔特色

### 1. 多層次說明

每個功能都包含四個層次的說明：

- **原理**：為什麼這樣設計，解決什麼問題
- **設計**：具體的實現方式和架構
- **使用情境**：何時應該使用這個功能
- **實際範例**：完整的使用範例和預期輸出

### 2. 繁體中文原生撰寫

本文檔完全使用繁體中文撰寫，而非機器翻譯：

- 符合繁體中文使用習慣的術語
- 清晰易懂的表達方式
- 適合華語開發者閱讀

### 3. 多 Agent 協作

本文檔由 7 個專門的 AI Agent 並行撰寫：

- 每個 Agent 專注於一個主題
- 確保每個章節的深度和專業性
- 統一的格式和風格
